Техническое задание: Универсальный парсер файлов
Цель проекта

Создать универсальный инструмент для парсинга, конвертации и анализа различных типов файлов, который будет полезен для разработчиков, аналитиков и энтузиастов данных.

Функциональные требования

Поддержка форматов файлов

Текстовые форматы: .txt, .md, .csv, .tsv

Табличные форматы: .xlsx, .ods

JSON и производные: .json, .geojson

XML и HTML: .xml, .html

Лог-файлы: .log

Конфигурационные: .ini, .yaml, .yml, .toml

Простые бинарные форматы: .bin (с возможностью hex-view)

Архивы: .zip, .tar, .gz (поддержка распаковки и парсинга содержимого)

Функции парсинга

Чтение и вывод структуры файлов в удобном формате (например, дерево или таблица).

Конвертация между форматами:

CSV ↔ Excel (.xlsx)

JSON ↔ YAML

HTML ↔ Markdown (текстовая часть)

Фильтрация данных по ключам/полям, регулярным выражениям или диапазону строк.

Поддержка пакетной обработки нескольких файлов одновременно.

Интерфейсы

CLI (Command Line Interface)

Команды типа:

parser parse --file data.csv --output data.json
parser convert --from json --to yaml --file config.json
parser analyze --file log.txt --filter "ERROR"

Библиотека для Python/Node.js

Позволяет использовать функции парсинга напрямую в коде.

Дополнительные функции

Автоматическое определение формата файла по расширению и содержимому.

Поддержка кодировок: UTF-8, UTF-16, ANSI/Windows-1251.

Логирование операций с выводом статистики (кол-во записей, пропущенных строк, ошибок).

Возможность расширения проекта плагинами для новых форматов.

Нефункциональные требования

Кроссплатформенность: Windows, macOS, Linux.

Минимальные внешние зависимости, предпочтительно использовать стандартные библиотеки + популярные open-source пакеты.

Легкий и читаемый код с документированными функциями.

Покрытие юнит-тестами хотя бы основных функций.
